3) History of AI

  • In 1950, people questioned themselves, "Will machines think like humans?"
  • Can machines learn?
  • Can they solve the problems?

b) Symbolic AI - (1960-1970):

  • Developers manually wrote the rules in the code
  • It is a rule-based approach

  • Symbolic AI has failed
  • They shifted from a rule-based approach to a learning-based approach

  • No need for developers to write the rules in the code
  • It automatically learns the pattern

e) Deep Learning - (present):

  • It uses a neural network to solve the complex problems

4) Kernel Method

  • Kernel Method is used for solving classification problems
  • The most well-known kernel method is SVM(Support Vector Machine)
  • The Main goal of the kernel method is to find the decision boundary

Decision Boundary:

  • It is a line or surface that divides the data into two categories

How does it work?

  • i)kernel Function:
    • kernel function is used to transform the data into high-dimensional data
  • ii)Kernel trick:
    • Kernel trick is used for non-linear separation of data
  • iii)Decision boundary:
    • The decision boundary is also called a hyperplane
    • Finally, it finds the hyperplane that divides the data into two categories

5) Probabilistic Modelling

  • Probabilistic Modelling uses probability and statistical methods to predict and understand the data

Key Concepts:

  • i)Bayes Theorem:
    • Bayes theorem is a mathematical formula used in probability and statistics
  • ii)Naive Bayes Classifier:
    • Naive Bayes Classifier is based on the supervised learning technique
    • It is used for solving both classification and regression problems
    • Naive Bayes classifier uses Bayes theorem to find the probability
  • iii)Logistic Regression:
    • Logistic Regression is based on the supervised learning technique
    • It is used for solving both classification and regression problems
    • Logistic Regression uses sigmoid functions to find the probability

Types of Probabilistic Modelling:

  • There are mainly three types. They are:
    • a)Generative Model
    • b)Discriminative Models
    • c)Graphical Models

7) state, state space, search tree, Search node

State:

  • State is the situation at a particular moment or time

State space:

  • State space is the collection of all states
  • It includes:
    • Initial State
    • Intermediate State
    • Goal State

Search tree:

  • Search tree is the tree representation of the state space
  • Each node in a tree is a state
    • a)root node -> Initial State
    • b)Intermediate node -> Intermediate state
    • c)leaves node -> Goal State

Search node:

  • A single element in the Search tree is called a Search node

9) Decision Tree

  • A Decision Tree is based on the Supervised Learning Technique
  • It is used for solving both classification and Regression Problems
  • A Decision Tree Consists of a Root Node, a Decision Node and a Leaf Node

Root Node:

  • Starting Node

Decision Node:

  • Make Decisions

Leaf Node:

  • Final Output

How does it work?

  • It splits the data based on the key features
  • It continues splitting and forms branches
  • It stops splitting until it reaches the final decision

10) Random Forest

  • Random Forest is based on the Supervised Learning Technique
  • It is used for solving both classification and Regression Problems
  • Random Forest combines multiple decision trees to predict the class of the dataset
  • Random Forest = Multiple Decision Trees + Ensembling

Ensembling:

  • It combines multiple outputs for better prediction
  • Because of this Ensemble Approach:
    • Random Forest reduces the overfitting
    • Improves accuracy

12) Gradient Boosting

  • Gradient Boosting is an Expansion of the Boosting Procedure
  • Gradient Boosting = Gradient Descent + Boosting

Boosting:

  • Boosting is based on the Ensemble Machine Learning Techniques
  • Boosting trains the decision trees sequentially and corrects the errors

Performance of Gradient Boosting:

  • i)Increase trees:
    • Increase the number of trees to give accurate predictions
  • ii)Reduce tree depth:
    • Decrease the tree depth to avoid overfitting
  • iii)Reduce Learning Rate:
    • Reducing the Learning Rate means the model learns slowly
    • It gives accurate predictions
